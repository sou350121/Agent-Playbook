# 04 — 範式轉變 (Paradigm Shifts)

> 每一次範式轉變都讓上一個時代的最佳實踐變成技術債。
> 本模塊幫你識別正在發生的轉變——在它變成共識之前。

---

## 本模塊定位

「範式」不是流行詞，而是描述一種**不可逆的思維方式切換**：
- 從「寫代碼」到「編排意圖」（Intent-Driven Development）
- 從「訓練模型」到「設計 Agent 行為」（Software 3.0）
- 從「個人技能」到「人機協作系統」

識別範式轉變的意義不是追潮流，而是**提前調整投資方向**。

## 核心範式：Intent-Driven Development

這是當前最重要的範式轉變，影響本 Playbook 的所有層次：

```
舊範式：
  工程師 → 寫語法 → 編譯器 → 運行

新範式：
  工程師 → 表達意圖 → Agent 編排 → 驗證結果
```

**Karpathy 的「Build with Intent」** 是這一轉變的旗幟性表述。
從「告訴機器怎麼做」到「告訴機器要達到什麼目標」——這不是工具升級，而是認知模型的根本性切換。
